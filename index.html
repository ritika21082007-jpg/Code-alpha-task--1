Secure Coding Review

Identifying Security Vulnerabilities and Secure Coding Practices

Project Overview

This project demonstrates a secure coding review of a JavaScript web application. It identifies security vulnerabilities and provides recommendations for safer coding.

Selected Programming Language & Application

Language: JavaScript

Application: User Login System

Vulnerable Code Example

const username = document.getElementById("user").value;

document.getElementById("output").innerHTML =
"Welcome " + username;

This code is vulnerable to Cross-Site Scripting (XSS) because it uses innerHTML.

Secure Code Example

const username = document.getElementById("user").value;

document.getElementById("output").textContent =
"Welcome " + username;

Using textContent prevents malicious script execution.

Security Vulnerabilities Found

Vulnerability Risk Description
Cross-Site Scripting (XSS) High Unsafe rendering of user input.
Weak Password Policy Medium Easy-to-guess passwords allowed.
Hardcoded Credentials Critical Sensitive data stored in source code.
Input Validation Issues High Unvalidated user data accepted.

Static Analysis Tools

  • ESLint Security Plugin
  • SonarQube
  • Semgrep
  • OWASP Dependency Check

Best Practices

  • Validate all user inputs.
  • Use HTTPS everywhere.
  • Store secrets securely.
  • Keep dependencies updated.
  • Implement strong authentication.
  • Apply least privilege principle.

Remediation Recommendations

Issue Solution
XSS Use textContent and sanitize inputs.
Weak Passwords Enforce strong password rules.
Hardcoded Credentials Use environment variables.
Input Validation Validate and sanitize all inputs.
